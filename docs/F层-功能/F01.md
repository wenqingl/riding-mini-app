# F01: 行者骑行数据合并工具 — 功能规格

## 功能组 A: 行者账号授权

### @FT-01 发起授权
```gherkin
Given 用户未登录
When 用户点击"登录行者"
Then 跳转行者 OAuth2 授权页面（client_id + redirect_uri + scope=write）
```
- 实现: `routers/auth.py::GET /auth/login` → `services/auth_service.py::get_auth_url()`

### @FT-02 授权回调处理
```gherkin
Given 用户在行者页面完成授权
When 行者回调到 redirect_uri 并携带 code
Then 后端用 code 换取 access_token 和 refresh_token
And 存储 token 到用户会话
```
- 实现: `routers/auth.py::GET /auth/callback` → `services/auth_service.py::exchange_code_for_token()`

### @FT-03 Token 刷新
```gherkin
Given access_token 已过期
When 用户发起操作
Then 后端使用 refresh_token 自动刷新 access_token
And 操作继续执行不中断
```
- 实现: `routers/auth.py::POST /auth/refresh` → `services/auth_service.py::refresh_access_token()`

### @FT-04 取消授权
```gherkin
Given 用户跳转到行者授权页面
When 用户点击取消
Then 回调返回错误码
And 小程序提示需要授权
```
- 实现: 小程序 `doLogin()` 模态框处理

## 功能组 B: 骑行记录管理

### @FT-05 获取记录列表
```gherkin
Given 用户已授权
When 进入主页
Then 调用行者 API 获取骑行记录列表
And 展示每条记录的日期、距离(km)、时长、起终点
```
- 实现: `routers/records.py::GET /api/records` → `services/record_service.py::get_records()`

### @FT-06 选择待合并的记录
```gherkin
Given 记录列表已展示
When 用户勾选记录（2条及以上）
Then 高亮选中项
And 显示"合并"按钮及选中数量
```
- 实现: 小程序 `index.js::toggleRecord()`

### @FT-07 取消选择
```gherkin
Given 用户已选中若干记录
When 用户取消勾选某条记录
Then 更新选中状态
If 选中数 < 2 则隐藏"合并"按钮
```
- 实现: 小程序 `index.js::toggleRecord()` + 条件渲染

## 功能组 C: 文件合并

### @FT-08 下载原始文件
```gherkin
Given 用户确认合并
When 系统调用行者 API 下载文件
Then 获取每条记录的 FIT/GPX/TCX 原始文件
```
- 实现: `services/record_service.py::download_record_file()`

### @FT-09 按时间顺序拼接
```gherkin
Given 所有原始文件已下载
When 系统解析并按时间戳排序
Then 按顺序拼接数据点，段间空白不补轨迹
And 合并总距离 = 各段距离之和
And 合并总时长 = 各段时长之和（不含空白间隔）
```
- 实现: `services/merge_service.py::merge_records()` + `records_to_gpx()`
- 关键逻辑: asyncio.gather 并行下载 → 按 timestamp 全局排序 → GPX 输出

### @FT-10 多格式统一
```gherkin
Given 选中记录包含不同格式
When 系统执行合并
Then 统一转换为目标格式后合并
And 保留心率、踏频、功率等传感器数据
```
- 实现: `services/merge_service.py::parse_fit_records()` / `parse_gpx_records()` / `parse_tcx_records()`

## 功能组 D: 上传 & 下载

### @FT-11 自动上传回行者
```gherkin
Given 合并文件已生成
When 系统调用行者上传接口
Then 上传成功，返回新记录 ID
And 小程序提示"上传成功，可在行者 App 查看"
```
- 实现: `routers/merge.py::POST /api/merge-and-upload` → `services/upload_service.py::upload_to_xingzhe()`

### @FT-12 下载到本地
```gherkin
Given 合并文件已生成
When 用户点击"下载"
Then 文件通过微信小程序下载能力保存到手机
```
- 实现: `routers/merge.py::POST /api/merge` (返回 GPX 二进制流) + 小程序 `api.js::mergeDownload()`

### @FT-13 上传失败重试
```gherkin
Given 行者上传接口返回错误
When 错误为临时性（网络超时、服务器5xx）
Then 自动重试最多3次
If 仍失败 则提示用户并提供手动下载选项
```
- 实现: `routers/merge.py::merge_and_upload()` — 上传失败时返回 `{success: false, merged_data_available: true}`
- 状态: MVP 阶段未做自动重试，降级为提供手动下载

## 行为规则

1. **最少记录数**: 合并操作至少需要 2 条记录
2. **Token 校验**: 所有 API 需 Bearer token，无效返回 401
3. **CSRF 防护**: OAuth state 使用随机值，回调时校验
4. **错误隔离**: 单条记录下载失败 → 整体返回 502；上传失败 → 降级为手动下载
5. **格式支持**: FIT / GPX / TCX 三种格式
