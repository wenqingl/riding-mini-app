# L01: 行者骑行数据合并工具 — 逻辑层规格

## 技术栈

- **后端**: Python 3.11+ + FastAPI
- **前端**: 微信小程序原生开发
- **FIT 解析**: python-fitparse
- **GPX/TCX 解析**: gpxpy / lxml
- **部署**: Nginx + HTTPS + Docker

## 测试基础设施

- **框架**: pytest + pytest-asyncio
- **命名约定**: `test_*.py`，按模块分目录
- **运行命令**: `pytest tests/ -v`
- **Mock**: unittest.mock (AsyncMock/MagicMock) 模拟行者 API
- **测试数量**: 23 个（全绿）

## 组件 1: Auth Service（OAuth 代理）

**职责**: 代理行者 OAuth 2.0 授权流程，管理 token

| 接口 | 方法 | 入参 | 出参 | 约束 |
|------|------|------|------|------|
| GET /auth/login | 重定向 | - | 302 → 行者授权页 | 拼接 client_id + redirect_uri + scope=write |
| GET /auth/callback | GET | code, state | {access_token, refresh_token} | 用 code 换 token，校验 state 防 CSRF |
| POST /auth/refresh | POST | refresh_token | {access_token} | token 过期时调用 |

**源码**: `services/auth_service.py`, `routers/auth.py`

## 组件 2: Record Service（骑行记录）

**职责**: 从行者 API 获取用户骑行记录列表

| 接口 | 方法 | 入参 | 出参 | 约束 |
|------|------|------|------|------|
| GET /api/records | GET | Bearer token | [{id, date, distance, duration, name}] | 带 Bearer token |
| GET /api/records/{id}/file | GET | Bearer token, id, format | FIT/GPX/TCX 二进制流 | 下载原始文件 |

**源码**: `services/record_service.py`, `routers/records.py`

## 组件 3: Merge Service（文件合并）

**职责**: 解析并合并多个运动文件

| 接口 | 方法 | 入参 | 出参 | 约束 |
|------|------|------|------|------|
| POST /api/merge | POST | {record_ids: [str], format: str} | GPX 二进制流 | 按时间排序拼接，至少 2 条记录 |
| POST /api/merge-and-upload | POST | {record_ids: [str], format: str} | {success, record_id, total_points} | 合并 + 自动上传 |

**源码**: `services/merge_service.py`, `routers/merge.py`

**核心逻辑**:
- `parse_fit_records()` — 解析 FIT 文件
- `parse_gpx_records()` — 解析 GPX 文件
- `parse_tcx_records()` — 解析 TCX 文件
- `merge_records()` — 多文件按时间戳合并（使用 asyncio.gather 并行下载）
- `records_to_gpx()` — 输出 GPX 格式

## 组件 4: Upload Service（上传回行者）

**职责**: 将合并后的文件上传回行者

| 接口 | 方法 | 入参 | 出参 | 约束 |
|------|------|------|------|------|
| POST /api/upload | POST | Bearer token, file, filename | {record_id, success} | 调用行者上传 API |

**源码**: `services/upload_service.py`

**注意**: 上传功能集成在 `/api/merge-and-upload` 接口中，`/api/upload` 为独立的上传端点。

## 组件 5: 公共工具

**职责**: 路由层共享工具函数

| 函数 | 所在文件 | 功能 |
|------|---------|------|
| `parse_token(authorization)` | `routers/utils.py` | 从 Authorization header 提取 Bearer token |
