# R01: 用户合并多日骑行记录

## 场景描述

用户进行了一次多日长途骑行，每天骑行结束后主动关闭码表，导致行程被拆分为多条独立记录。用户希望将这些记录合并为一条完整轨迹，以展示完整的骑行路线和累计数据。

## AT 规格

### @AT-01 用户授权登录并查看骑行记录列表
```gherkin
Given 用户已安装行者 App 并有骑行记录
When 用户打开小程序并点击"登录行者"
Then 跳转行者 OAuth 授权页面
When 用户完成授权
Then 小程序显示用户的骑行记录列表（含日期、距离、时长）
```
- 测试方法: `test_auth.py::test_get_auth_url_contains_client_id` + 小程序手动测试

### @AT-02 用户选择多条记录进行合并
```gherkin
Given 用户已登录且记录列表已加载
When 用户勾选3条骑行记录（日期连续）
And 点击"合并"按钮
Then 系统下载这3条记录的 FIT/GPX/TCX 文件
And 按时间顺序拼接为一个文件（段间空白不补轨迹）
And 生成合并后的运动数据文件
```
- 测试方法: `test_merge.py::test_merge_two_files` + 小程序手动测试

### @AT-03 合并后的文件自动上传回行者
```gherkin
Given 合并文件已生成
When 系统调用行者上传接口
Then 文件成功上传到用户行者账号
And 用户在行者 App 中可以看到合并后的新记录
```
- 测试方法: `test_merge_and_upload.py::test_merge_and_upload_success`

### @AT-04 合并后的文件支持下载到本地
```gherkin
Given 合并文件已生成
When 用户点击"下载"按钮
Then 文件保存到用户手机本地
```
- 测试方法: `test_merge.py::test_records_to_gpx_roundtrip` + `test_merge_and_upload.py::test_merge_only_returns_gpx`

### @AT-05 行者授权失败的处理
```gherkin
Given 用户打开小程序
When 用户点击"登录行者"但取消授权
Then 小程序提示"需要授权才能使用"并引导重新授权
```
- 测试方法: 小程序手动测试

### @AT-06 合并记录格式不一致的处理
```gherkin
Given 用户选中的记录包含不同格式（如 FIT 和 GPX）
When 系统执行合并
Then 统一转换为目标格式后合并
And 保留所有有效的运动数据点
```
- 测试方法: `test_merge.py` — GPX 解析 + 合并测试（TCX 解析代码已实现，依赖 lxml）

## 约束

- 先只支持行者，不扩展其他平台
- 段间空白直接拼接，不补轨迹
- 广告模式延后，先做功能
